name: project-impact-analysis
description: Map verified evidence and clusters to project-specific impact.
---

## When to use

Use after evidence synthesis for routes requiring `impact`.

## Inputs

Project profile, evidence results, related clusters, current events.

## Outputs

Affected modules, semantic relevance, risk, related events, conflicts.

## Steps

1. Match evidence to critical modules and dependencies.
2. Account for source diversity and conflicting claims.
3. Explain semantic relevance and risk.

## Constraints

Never emit or override authoritative `final_score`.
